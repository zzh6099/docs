# 组件

# 介绍

组件是用于构建组件实例的函数，它带有一个名字：在下面例子中是 <ButtonCounter>。我们在一个根实例中，把这个组件作为自定义元素来使用。你可以将组件进行任意次数的复用：

function ButtonCounter() {
  return {
    data: function(){
      return {
        count: 0
      }
    },
    template: '<button @click="counter">你点击了 {{ count }} 次.</button>',
    methods: {
      counter: function () {
        this.$setData("count", this.count + 1)
      }
    }
  }
}

toolkit.createApp({
  template:
    '<div id="components-demo">\
      <ButtonCounter></ButtonCounter>\
      <ButtonCounter></ButtonCounter>\
      <ButtonCounter></ButtonCounter>\
    </div>',
  components: {
    ButtonCounter: ButtonCounter
  },
  el: "#app"
})

每次使用组件，都会各自独立维护它的 count。因为你每用一次组件，就会有一个它的新实例被创建。

组件与根实例接收相同的选项，例如 data、computed、watch、methods 以及生命周期钩子等。仅有的例外是像 el 这样根实例特有的选项。

为了能在模板中使用，组件必须先注册以便能够被识别。

与组件相关的 选项 请参考：

  • 数据 - props

# 注册局部组件

使用 components 选项注册局部组件。components 选项接收一个对象，对象的 key 会作为局部组件的名称，在 template 模板中通过局部组件的名称使用局部组件。

 








 

 
 
 



function component() {
  return {
     template: '<div>被注册为局部组件</div>'
  }
}

toolkit.createApp({
  template:
    '<div id="components-demo">\
      <LocalComponent></LocalComponent>\
    </div>',
  components: {
    LocalComponent: component
  },
  el: "#app"
})

# 注册全局组件

使用全局函数 registerComponent 注册全局组件，在 template 模板中可以直接使用全局组件。

  • 参数：

    • {string} name - 全局组件的名称。
    • {Function} component - 组件。
    • {Object} [options] - 可选，额外的注册选项。

  • 详细：

     








     




    registerComponent("GlobalComponent", function () {
  return {
    template: '<div>被注册为全局组件</div>'
  }
})

toolkit.createApp({
  template:
    '<div id="components-demo">\
      <GlobalComponent></GlobalComponent>\
    </div>',
  el: "#app"
})

    通过使用 registerComponent 全局注册了 <GlobalComponent> 组件。从而可以在根实例的模板中直接使用。

# 组件库

组件库是一个包含若干组件的 js 文件，其中导出的组件，可通过局部注册导入使用。

# 导出组件

通过全局函数 exportComponent 导出组件。

  • 参数：

    • {string} name - 组件的名称。
    • {Function} component - 组件。
    • {Object} [options] - 可选，额外的注册选项。

  • 详细：

    exportComponent("Component",function () {
  return {
      template: '<div>被导出的组件</div>'
  }
})

    组件库导出的所有组件，会存放到以组件库的 URL 创建的命名空间中。同一个组件库内的各个组件，组件名称不能重复，而不同的组件库的组件由于存放在不同的命名空间，所以可以使用相同的组件名称。

# 导入组件

通过全局函数 importComponent 导入组件。导入的组件只能局部注册。

  • 参数：

    • {string} name - 要导入的组件的名称。

    • {string}} URL - 组件库的 URL。

    • {Object} [options] - 可选，选项对象。可以使用以下选项：

      • loading：Component | Element | String
        异步组件加载期间，显示的 Loading 组件。支持组件或 DOM 元素，全局组件使用组件名称。

      • error：Component | Element | String
        异步组件加载失败时，显示的 Error 组件。支持组件或 DOM 元素，全局组件使用组件名称。

      • timeout：Number
        异步组件加载超时毫秒数，默认 0。

      • lazy：Boolean
        设置为 true 时，异步组件采用懒加载，当组件要被渲染时才开始加载。默认 false。

      • loadingDelay：Number
        Loading 组件延迟显示的毫秒数，默认 200。

      • loadingMinDuration：Number
        Loading 组件的最少显示时长，默认 200。

  • 详细：




     


     




    toolkit.createApp({
  template:
    '<div id="components-demo">\
      <MyComponent></MyComponent>\
    </div>',
  components: {
    MyComponent: importComponent("Component", "/components.js")
  },
  el: "#app"
})

    导入组件可以是同步导入，也可以异步导入。同步导入要在 html 中使用 script 标签预加载组件库。当没有同步导入时，会在调用 importComponent 时自动发起请求，异步加载组件库。
