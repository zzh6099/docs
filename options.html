# 选项

# DOM

# el

  • 类型：string | Element

  • 限制：只在创建根实例时生效。组件不需要该选项。

  • 详细：

    该选项用于定义根实例的挂载目标。

    提供一个在页面上已存在的 DOM 元素，也可以是 CSS 选择器。可以用 $el 访问该挂载目标元素。

# template

  • 类型：string

  • 详细：

    该选项用于定义字符串模板。

    可以为 template 选项直接提供一个字符串模板。

    也可以为 template 选项提供一个 #id，并在 HTML 中使用 <script type="template" id="xxx">...</script> 来定义模板。

    模板生成的 DOM 将会插入挂载目标 el 元素中。

    模板支持多种渲染指令，事件绑定指令及内置的组件等，具体请查看模板。

    模板支持多个根元素。

  • 示例：

    直接提供一个字符串模板。推荐使用【Tab】缩进、【\】换行，提高模板可读性。

    toolkit.createApp({
    template:
      '<div>\
          根元素1\
          <div>template</div>\
      </div>\
      <div>根元素2</div>',
    el: "#app"
})

    提供一个 #id，并在 HTML 中提供一个字符串模板。

    <script type="template" id="my-template">
  <div>
      根元素1
      <div>template</div>
  </div>
  <div>根元素2</div>
</script>

    toolkit.createApp({
    template: "#my-template",
    el: "#app"
})

# 数据

注意

data、props、computed、methods 不能定义相同的名称。

# data

  • 类型：Function

  • 详细：

    该选项用于定义数据对象。

    data 是一个返回对象的函数。返回的对象必须是纯粹的对象 (含有零个或多个的 key/value 对)，浏览器 API 创建的原生对象，原型上的属性会被忽略。

    在创建实例之前，应该在 data 中声明所有实例用到的数据。

    函数的上下文对象 this，为组件实例对象，可以访问到 props 及状态管理工具 store 的数据。

  • 示例：




     
     
     
     
     
     









    function data_demo() {
  return {
      template: '<div>{{ key }} {{ title }}</div>',
      data: function () {
          return { 
            key: this.propKey + 1,
            title: ""
          }
      },
      props: {
          propKey: {
              type: Number,
              required: true
          }
      }
  }
}

# props

  • 类型：Array<string> | Object

  • 详细：

    该选项用于定义接收到的来自父组件的数据。

    props 可以是简单的数组，或者使用对象，对象允许配置高级选项，如类型检测、自定义验证和设置默认值。

    你可以基于对象的语法使用以下选项：

    • type： 可以是下列原生构造函数中的一种：String、Number、Boolean、Array、Object、Date、Function、Symbol、任何自定义构造函数、或上述内容组成的数组。会检查一个 prop 是否是给定的类型，否则抛出警告。
    • default：any
      为该 prop 指定一个默认值。如果该 prop 没有被传入，则换做用这个值。
    • required：Boolean
      定义该 prop 是否是必填项。如果这个值为 “真” 且该 prop 没有被传入的，则一个控制台警告将会被抛出。
    • validator：Function
      自定义验证函数会将该 prop 的值作为唯一的参数代入。如果该函数返回一个 “假” 的值 (也就是验证失败)，一个控制台警告将会被抛出。

  • 示例：





     










     

     



     




     





     
     
     



     
     
     
     






    // 简单语法
function props_demo_simple() {
  return {
    ...
    props: ['size', 'myMessage'],
    ...
  }
}

// 对象语法，提供验证
function props_demo_advanced() {
  return {
    ...
    props: {
      // 基础的类型检查 (`null` 和 `undefined` 会通过任何类型验证)
      propA: Number,
      // 多个可能的类型
      propB: [String, Number],
      // 必填的字符串
      propC: {
        type: String,
        required: true
      },
      // 带有默认值的数字
      propD: {
        type: Number,
        default: 100
      },
      // 带有默认值的对象
      propE: {
        type: Object,
        // 对象或数组默认值必须从一个工厂函数获取
        default: function () {
          return { message: 'hello' }
        }
      },
      // 自定义验证函数
      propF: {
        validator: function (value) {
          // 这个值必须匹配下列字符串中的一个
          return ['success', 'warning', 'danger'].indexOf(value) !== -1
        }
      }
    },
    ...
  }
}

# computed

  • 类型：{ [key: string]: Function }

  • 详细：

    该选项用于定义计算属性。

    模板内的表达式非常便利，但是设计它们的初衷是用于简单运算的。在模板中放入太多的逻辑会让模板过重且难以维护。例如：

    <div id="example">
  {{ message.split('').reverse().join('') }}
</div>

    在这个地方，模板不再是简单的声明式逻辑。你必须看一段时间才能意识到，这里是想要显示变量 message 的翻转字符串。当你想要在模板中的多处包含此翻转字符串时，就会更加难以处理。

    所以，对于任何复杂逻辑，你都应当使用计算属性。

    计算属性的结果会被缓存，除非依赖的数据变化才会重新计算。

    计算属性将被混入到实例中，可以像 data 选项中的属性那样使用，除了不能用 $setData 修改。

  • 示例：













     
     
     
     
     



    toolkit.createApp({
  template:
    '<div id="example">\
      <input $model="message">\
      <p>message: "{{ message }}"</p>\
      <p>reversed message: "{{ reversedMessage }}"</p>\
    </div>',
  data: function () {
    return {
      message: 'Hello'
    }
  },
  computed: {
    reversedMessage: function () {
      return this.message.split('').reverse().join('')
    }
  },
  el: "#app"
})

# methods

  • 类型：{ [key: string]: Function }

  • 详细：

    该选项用于定义实例的方法。

    这些方法将被混入到实例中。可以直接通过 this 调用这些方法，或者在模板的指令、数据绑定及事件绑定中使用，也可作为 prop 传递给子组件使用。

    注意：不应该使用箭头函数来定义这些方法 (例如 plus: () => this.a++)。理由是箭头函数绑定了父级作用域的上下文，所以 this 将不会按照期望指向实例，this.a 将是 undefined。

  • 示例：











     
     
     
     
     
     
     
     



    toolkit.createApp({
  template:
    '<button @click="plus" style="padding:5px 20px;">\
      {{ double() }}\
    </button>',
  data: function(){
    return {
      a: 1
    }
  },
  methods: {
    plus: function () {
      this.$setData("a", this.a + 1)
    },
    double: function () {
      return this.a * 2
    }
  },
  el: "#app"
})

# watch

  • 类型：{ [key: string]: string | Function | Object }

  • 详细：

    该选项用于观察数据变化，并在变化时执行回调函数。

    可以观察 data、props、computed 内的数据。

    定义为一个对象，键是需要观察的数据，支持使用键路径观察对象或数组内部的数据，值是对应回调函数。值也可以是 methods 中的方法名，或者是包含选项的对象。

    你可以基于对象的语法使用以下选项：

    • handler：Function
      数据变化后执行的回调函数，其得到的参数为数据的旧值和新值。

    • immediate：Boolean
      回调函数将会在侦听开始之后被立即调用。

    • deep：Boolean
      回调函数会在任何被侦听的对象的属性改变时被调用，不论其被嵌套多深。

  • 示例：


























     
     
     
     

     


     
     
     
     



     
     





    toolkit.createApp({
  template:
    '<p>{{ a }}</p>\
    <p>{{ b }}</p>\
    <p>{{ c }}</p>\
    <p>{{ d }}</p>\
    使用下面的编辑器修改数据，看看效果',
  data: function(){
    return {
      a: 1,
      b: 2,
      c: 3,
      d: {
        e: {
          f: 5
        }
      }
    }
  },
  methods: {
    aPlus: function () {
      this.$setData("a", this.a + 1)
      alert("属性 a 的值加 1")
    }
  },
  watch: {
    a: function (oldVal, newVal) {
      alert(`监听到属性 a 更新，旧值：${oldVal}，新值：${newVal}`)
    },
    // 直接使用 methods 中的方法
    b: 'aPlus',
    // 回调函数将会在侦听开始之后被立即调用
    c: {
      handler: function (oldVal, newVal) {
        console.log('侦听开始之后被立即调用')
      },
      immediate: true
    },
    // 侦听d.e，该回调会在被侦听的对象的任何属性改变时被调用，不论其被嵌套多深
    'd.e': {
      handler: 'aPlus',
      deep: true
    }
  },
  el: "#app"
})

    提示

    框架内部会检测状态变更是否陷入死循环，当发现由一次状态变更引发的一系列状态变更超过 100 次，且循环变更执行超过 10 组时，会认为触发了死循环而报错并中断变更，报错信息会提示死循环的循环变更路径。

    在下面的示例中，当点击文本后，会触发状态 count 变更，然后 computed 的 double 和 plus 相继变更，最后 watch 中监听 double 的 hanlder 函数被调用后，再次触发 count 变更，形成了死循环。

    toolkit.createApp({
  template:
    '<div @click=`$setData("count",0)`>count:{{count}} double:{{double}} plus:{{plus}}</div>',
  data:function(){
    return {
      count:1
    }
  },
  computed:{
    "double":function(){
      return this.count*2
    },
    "plus":function(){
      return this.double+1
    }
  },
  watch:{
    "double":function(){
        this.$setData("count",this.count+1)
    }
  },
  el:"#app"
})

# emits

  • 类型：Array<string> | Object

  • 详细：

    emits 用于声明该组件触发的自定义事件，这些事件不会被组件的根元素继承，也不会出现在 $attrs 中。

    emits 可以是数组或对象。在对象语法中，每个属性的值可以为 null 或验证函数。验证函数将接收传递给 $emit 的事件参数（除第一个参数外的所有参数）。如果 this.$emit('foo',1) 被调用，foo 的相应验证函数将接收参数 1。验证函数应返回布尔值，以表示事件参数是否有效。

    没有在 emits 中声明的自定义事件，仍然可以使用 $emit 触发，但这些事件会出现在 $attrs 中，且可以被组件的根元素继承，。

  • 示例：

    数组语法：



     






    function emits_demo_component() {
  return {
    emits: ["check"],
    mounted() {
      this.$emit('check')
    }
  }
}

    对象语法：





     


     
     
     
     
     
     
     
     







    function emits_demo_component() {
  return {
    emits: {
      // 没有验证函数
      click: null,

      // 带有验证函数
      submit: payload => {
        if (payload.email && payload.password) {
          return true
        } else {
          console.warn(`无效的 submit 事件参数`)
          return false
        }
      }
    },
    mounted() {
      this.$emit('submit', { email: "xxx", password: "xxx" })
    }
  }
}

# 资源

# mixins

  • 类型：Array<Object>

  • 详细：

    mixins 选项接收一个混入对象的数组。这些混入对象可以像正常的组件一样包含选项，这些选项将会被合并到组件的选项中。

    当组件和混入对象含有同名选项时，这些选项将以恰当的方式进行“合并”。合并逻辑，如下：

    • data - 数据对象在内部会进行浅层合并（只合并根级属性），并在发生冲突时以组件数据优先。
    • props、computed、methods、emits - 将合并为同一个对象，键名冲突时，值会以组件优先。
    • watch - 将合并为同一个对象，键名冲突时，值会被合并为数组。另外，混入对象的 handler 将在组件的 handler 之前调用。
    • 生命周期钩子 - 同名钩子函数将合并为一个数组，因此都将被调用。另外，混入对象的钩子将在组件自身钩子之前调用。
    • 其它 - 不会合并。

  • 示例：
























     












    var mixin = {
    data: function () {
        return {
            title: "title",
            name: "mixin_name",
            user: {
                name: "John",
                age: 16
            }
        }
    },
    methods: {
        upper: function (val) {
            return val.toUpperCase()
        }
    }
}

toolkit.createApp({
    template:
        '<div>title: {{ upper(title) }}</div>\
        <div>name: {{ name }}</div>\
        <div>user: {{ user }}</div>',
    mixins: [mixin],
    data: function () {
        return {
            name: "name",
            user:{
              name: "Tom",
              sex: "man"
            }
        }
    },
    el: "#app"
})

    如果混入对象也有 mixins 选项时，在合并时会进行递归合并。

# components

  • 类型：{ [key: string]: Component }

  • 详细：

    该选项用于定义要使用的局部组件。

    components 对象的 key 会作为临时组件名称，在 template 中使用临时组件名称引用局部组件。更多关于组件的信息，请查看组件

# 生命周期钩子

# beforeMount

  • 类型：Function

  • 详细：

    在实例的 DOM 被挂载前调用。

# mounted

  • 类型：Function

  • 详细：

    在实例的 DOM 被挂载后调用。可以在这个钩子执行依赖于实例的 DOM 的操作。

    mounted 不会保证所有的子组件也都被挂载完成。

# beforeUpdate

  • 类型：Function

  • 详细：

    在数据发生改变后，实例的 DOM 被更新之前被调用。

# updated

  • 类型：Function

  • 详细：

    在数据发生改变后，实例的 DOM 更新完毕之后调用。可以在这个钩子执行依赖于实例的 DOM 的操作。

    updated 不会保证所有的子组件也都被重新渲染完毕。

    注意

    不要在这个钩子执行无条件的变更数据操作，如：$setData，$store.commit 等，否则可能会导致死循环。

# beforeUnmount

  • 类型：Function

  • 详细：

    在实例销毁之前调用。在这一步，实例仍然完全可用。

# unmounted

  • 类型：Function

  • 详细：

    在实例销毁后调用。该钩子被调用后，实例的所有事件监听器被移除，所有的子组件也都被销毁。

# 其它选项

# inheritAttrs

  • 类型：Boolean

  • 默认值：true

  • 详细：

    当组件接收到的某些属性或事件，没有在组件的 props 或 emits 中定义时，会被组件收集起来。常见的示例包括 class、style 和 id 属性。

    默认情况下，这些收集的属性或事件，将被根节点自动继承，例如，在 <datePicker> 组件的示例中：

    function datePicker() {
  return {
    template: `
      <div class="date-picker">
        <input type="datetime-local" />
      </div>
    `
  }
}

    如果我们在使用 <datePicker> 组件时，添加 data-status 属性，由于没有在组件的 props 中定义，它将自动被根节点继承 (即 div.date-picker)。

    <datePicker data-status="activated"></datePicker>

<!-- datePicker 组件渲染结果 -->
<div class="date-picker" data-status="activated">
  <input type="datetime-local" />
</div>

    同样的规则也适用于事件监听器。

    如果你不希望组件的根节点自动继承它们，你可以设置 inheritAttrs: false。这时，你可以通过组件的 $attrs 属性访问它们。并可以使用 $bind 指令，把它们应用于其他节点。








     



    function datePicker() {
  return {
    template: `
      <div class="date-picker">
        <input type="datetime-local" $bind="$attrs"/>
      </div>
    `,
    inheritAttrs: false
  }
}

    现在 data-status 属性将应用于 input 元素！

    <datePicker data-status="activated"></datePicker>

<!-- datePicker 组件渲染结果 -->
<div class="date-picker">
  <input type="datetime-local" data-status="activated" />
</div>

    注意

    在 Vue 3 中，若组件返回多个根节点，你需要使用 v-bind="$attrs" 来显式声明 $attrs 绑定到哪个节点，否则 Vue 会发出运行时警告。

    但在本框架中，当返回多个根节点时，每个根节点都会自动继承，当你有明确要指定某个节点继承的需求时，可以使用 $bind 指令绑定 $attrs 到该节点。

# comments

  • 类型：Boolean

  • 默认值：false

  • 详细：

    当设为 true 时，将会保留且渲染 template 模板中的 HTML 注释。默认行为是舍弃它们。

# style

  • 类型：Object

  • 详细：

    style 属性通过使用一个样式对象来描述组件的私有样式。

    选择器样式：

    在样式对象中，使用 css 选择器作为键 ，值为一个对象，对象包含作用于该选择器的样式属性：

    {
  style: {
    ".red": {
      "color": "red"
    }
  }
}

    支持选择器嵌套语法。选择器嵌套指的是在一个选择器中嵌套另一个选择器来实现继承，从而增强了样式的结构性和可读性。在选择器嵌套中，可以使用 “&” 表示父元素选择器：

    toolkit.createApp({
  template:
    '<p class="red">红色字体</p>\
    <div class="outter">\
      <div class="inner">灰色背景</div>\
    </div>\
    <div class="outter bg-green">\
      <div class="inner">绿色背景</div>\
    </div>',
  el: "#app",
  style: {
    ".red": { //此处编译为 .red {color:red;}
      "color": "red"
    },
    ".outter": { //此处编译为 .outer {background-color:grey;}
      "background-color": "grey",
      
      ".inner": { //此处嵌套语法编译为 .outer .inner {text-align:center;}
        "text-align": "center"
      },
      "&.bg-green": { //此处嵌套语法编译为 .outer.bg-green {background-color:green;}
        "background-color": "green",
      },
      "&:before": { //此处嵌套语法编译为 .outter:before {content:"before";font-size:16px;}
        "content": "before",
        "font-size": "16px",
        "text-align": "center",
        "display": "block"
      }
    }
  }
})

    动画 @keyframes：

    在样式对象中，使用 @keyframes 为键，值为一个对象，该对象可以包含多个动画定义：























     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     
     




    toolkit.createApp({
  template:
    '<div>\
      <button @click=`$setData("show",!show)`>Toggle show</button>\
      <transition name="bounce">\
        <p $if="show">\
          Lorem ipsum dolor sit amet, consectetur adipiscing elit. Mauris facilisis enim libero, at lacinia diam fermentum id. Pellentesque habitant morbi tristique senectus et netus.\
        </p>\
      </transition>\
    </div>',
  data: function () {
    return {
      show: true
    }
  },
  style: {
    ".bounce-enter-active": {
      "animation": "bounce-in .5s"
    },
    ".bounce-leave-active": {
      "animation": "bounce-out 1s"
    },
    "@keyframes": {
      "bounce-in": {
        "0%": {
          "transform": "scale(0)"
        },
        "50%": {
          "transform": "scale(1.5)"
        },
        "100%": {
          "transform": "scale(1)"
        }
      },
      "bounce-out": {
        "0%": {
          "transform": "translateX(0) scale(1)"
        },
        "30%": {
          "transform": "translateX(0) scale(1.5)"
        },
        "100%": {
          "transform": "translateX(100%) scale(0)"
        }
      }
    }
  },
  el: "#app"
})
